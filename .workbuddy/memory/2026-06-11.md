# 2026-06-11 立绘系统重写

## 完成的工作

### 重写目的
konado 插件默认立绘是单纹理，Clara 是16层组合立绘系统。之前用 Viewport + SubViewport 做了初步改造但存在大量问题：
- Clara 和 Eve 走两条完全独立的 Actor 路径
- project_actor_layout.tres 的 slot_y_offset 与 Clara 内部 _apply_layout 互相覆盖
- 交叉溶解过渡重复实现在两个子类中
- Viewport 初始化时序脆弱

### 新架构

```
PortraitActorBase (extends KND_Actor)
  ├── SimplePortraitActor  → Eve 等单图角色
  └── CompositePortraitActor → Clara 组合立绘
```

核心原则：
1. 所有立绘统一走 SubViewport 架构
2. 布局参数集中在 PortraitActorLayoutDB
3. 交叉溶解管道在基类一次实现（fade_apply_state）
4. 子类只需实现 _render_frame_to_viewport() 和 apply_state()

### 第六轮重构（去掉 SVC，改用 TextureRect + VP）

**架构变更：**
- 删除 SubViewportContainer，改为 TextureRect 直接显示 VP 纹理
- 布局极简：`_texture_rect.scale = Vector2(config_scale, config_scale)`，不用任何屏幕尺寸推导
- `_texture_rect.position = 底部居中于 Slot`
- 交叉溶解：双 TextureRect 叠放 alpha 渐变（`_overlay_rect`），不再需要 blend shader
- 删除 `portrait_blend.gdshader`
- 清理 `project_actor_layout.tres` 和 `portrait_actor_layout_db.gd` 中的 `container_fit` 字段
- SimplePortraitActor: `_texture_rect` → `_inner_rect`（避免与基类 display TextureRect 重名）
- CompositePortraitActor: 移除 `_get_content_aspect()` 死代码

### 注意
新建的 tscn 文件无 .uid（需 Godot 打开项目后自动生成）。
layout_config 在 add_child 之前注入，确保 _ready() 可读取。
